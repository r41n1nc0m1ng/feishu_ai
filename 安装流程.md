# 本地部署安装流程

## 前置条件

### 1. Neo4j Desktop

1. 前往 [neo4j.com/download](https://neo4j.com/download/) 下载并安装 Neo4j Desktop
2. 打开 Neo4j Desktop → **New Project** → **Add** → **Local DBMS**
3. 设置数据库密码（默认填写 `password`，与 `.env` 保持一致）
4. 点击 **Start** 启动数据库
5. 确认状态栏显示 **Active**，连接地址为 `neo4j://localhost:7687`

### 2. Ollama

1. 前往 [ollama.com](https://ollama.com) 下载并安装 Ollama
2. 安装完成后拉取所需模型：

```bash
ollama pull qwen2.5:7b        # 主力推理模型
ollama pull nomic-embed-text   # 向量嵌入模型
```

3. 验证模型已安装：

```bash
ollama list
```

输出中应包含 `qwen2.5:7b` 和 `nomic-embed-text`。

### 3. Python 环境

要求 Python 3.10 及以上。

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate

pip install -r requirements.txt
```

## 飞书开发者配置

> 以下操作在 [open.feishu.cn/app](https://open.feishu.cn/app) 完成。

### 1. 创建应用

1. 点击 **创建企业自建应用**
2. 填写应用名称（例如 `OpenClaw Memory`）和描述
3. 记录页面上的 **App ID** 和 **App Secret**

### 2. 开启机器人能力

左侧菜单 → **应用能力** → **机器人** → 开启

### 3. 配置事件订阅（长连接模式）

左侧菜单 → **事件与回调** → **事件配置**

- **接收事件方式** 选择 **长连接（WebSocket）**（无需公网地址）
- 点击 **添加事件** → 搜索并添加 `im.message.receive_v1`

### 4. 发布应用

左侧菜单 → **版本管理与发布** → **创建版本** → 提交审核

## 项目配置

复制环境变量模板并填写：

```bash
cp .env.example .env
```

编辑 `.env`：

```env
# 飞书应用凭据（从开发者控制台复制）
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# OpenClaw 服务地址（留空则回退到本地 Ollama）
OPENCLAW_URL=

# Ollama 本地地址
OLLAMA_URL=http://localhost:11434
LOCAL_MODEL=qwen2.5:7b
EMBED_MODEL=nomic-embed-text

# Neo4j（与 Neo4j Desktop 设置保持一致）
NEO4J_URI=neo4j://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password
```

## 启动机器人

确保 Neo4j Desktop 数据库已启动、Ollama 正在运行，然后：

```bash
python main.py
```

启动成功的标志：

```
INFO:Lark:connected to wss://msg-frontier.feishu.cn/ws/v2
INFO:__main__:Bot is running. Press Ctrl+C to stop.
INFO:memory.graphiti_client:Graphiti ready (LLM=qwen2.5:7b, embed=nomic-embed-text)
```

## 常见问题

### Ollama 请求失败（503 / Connection refused）

- 确认 Ollama 正在运行：`ollama list`
- Windows 用户：系统代理可能拦截 localhost 请求，代码中已通过 `trust_env=False` 处理，如仍有问题请关闭系统代理后重试

### Neo4j 连接失败（DatabaseNotFound / ServiceUnavailable）

- 确认 Neo4j Desktop 中数据库状态为 **Active**
- `.env` 中使用 `neo4j://localhost:7687`（而非 `bolt://`），这是 Neo4j Desktop 的路由协议地址

### 飞书消息发送后机器人无响应

1. 确认开发者控制台 → 事件配置 → 接收方式已设置为**长连接**
2. 确认已添加 `im.message.receive_v1` 事件订阅
3. 确认机器人已被添加到目标群聊
4. 查看终端日志，`connected to wss://...` 表示 WebSocket 已建立

### 首次启动出现大量 INFO 日志（Neo4j 索引）

属于正常现象，仅首次建立索引时出现，后续启动不再重复。
